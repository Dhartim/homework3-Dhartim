Geospatial Data Visualization

Introduction

This homework focus on developing a Geospatial data visualization. Used a non-proportional Symbol Map data visualization technique

Data

For this visualization data from the SF 311 Cases dataset. It is a portal where complaints are registered. The City and County of San Francisco established SF311 to provide an easy-to-remember telephone number that connects residents, businesses, and visitors to Customer Service Representatives ready to help with general government information and services. The dataset was downloaded on 30th March, 2020.

"This data is made available under the Public Domain Dedication and License v1.0 whose full text can be found at: www.opendatacommons.org/licenses/pddl/1.0/"

Here is the source Link

The dataset comes from DataSF. It contains data about 311 cases which are opened in the month of February, 2020. Info about the dataset from the site are as follows:

678 Rows 20 Columns 283 KB CSV file size
  1. CaseID: The unique ID of the service request created.
  2. Opened: The date and time when the service request was made
  3. Closed: The date and time when the service request was closed
  4. Updated: The date and time when the service request was last modified. For requests with status=closed, this will be the date the request was closed
  5. Status: A single-word indicator of the current state of the service request. (Note: GeoReport V2 only permits ‘open’ and ‘closed’)
  6. Status Notes: Explanation of why status was changed to current state or more details on current status than conveyed with status alone
  7. Responsible Agency: The agency responsible for fulfilling or otherwise addressing the service request
  8. Category: The human readable name of the service Category
  9. Request Type: The human readable name of the service request type
  10. Request Details: The human readable name of the service request details
  11. Supervisor District: San Francisco Supervisor District as defined in 'Supervisor Districts as of April 2012'
  12. Neighborhoods: San Francisco Neighborhood as defined in 'SF Find Neighborhoods'
  13. Police District: San Francisco Police District as defined in 'Current Police Districts'
  14. Latitude: Latitude of the location, using the WGS84 projection
  15. Longitude: Longitude of the location, using the WGS84 projection
  16. Point: Combination of Latitude and Longitude for Socrata native maps
  17. Source: Mechanism or path by which the service request was received; typically ‘Phone’, ‘Text/SMS’, ‘Website’, ‘Mobile App’, ‘Twitter’, etc but terms may vary by system.
